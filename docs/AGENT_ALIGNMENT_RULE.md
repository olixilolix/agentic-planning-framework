# Agent Alignment Rule

## Purpose

This document keeps `AGENTS.md` derived from the repo's planning trunk instead of becoming a second hidden architecture.

## Core Rule

If `AGENTS.md` changes meaningfully, update the relevant planning docs too.

If the planning docs change meaningfully, update `AGENTS.md` if the change affects agent behavior.

## Conflict Rule

If `AGENTS.md` and the planning trunk disagree:

1. the planning trunk wins
2. the mismatch should be fixed explicitly
3. the repo should not proceed by pretending both are fine

## Working Rule

Update both or neither.
