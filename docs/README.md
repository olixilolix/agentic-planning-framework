# Framework Docs

This folder contains the reusable workflow and governance spine of the Agentic Planning Framework Kit.

These docs are intentionally generic. They should tell a repo how to operate work without becoming a substitute for that repo's product architecture.

## File Guide

- [DEVELOPMENT_FRAMEWORK.md](DEVELOPMENT_FRAMEWORK.md): canonical workflow model and the `Frame -> Lock -> Slice -> Build -> Verify -> Close` sequence
- [DECISION_GATES.md](DECISION_GATES.md): what an agent may assume safely versus what must be escalated
- [PRE_PHASE_QA_PROTOCOL.md](PRE_PHASE_QA_PROTOCOL.md): bounded questions to answer before a new milestone or ticket cluster
- [PHASE_CLOSE_CHECKLIST.md](PHASE_CLOSE_CHECKLIST.md): closeout protocol before saying meaningful work is complete
- [TEST_CHARTER.md](TEST_CHARTER.md): quality bar by layer
- [DONOR_TRIAGE_CHECKLIST.md](DONOR_TRIAGE_CHECKLIST.md): gate for adopting external patterns or ideas
- [AGENT_ALIGNMENT_RULE.md](AGENT_ALIGNMENT_RULE.md): rule that keeps `AGENTS.md` aligned with the planning trunk
- [PHASE_OPERATING_PLAYBOOK.md](PHASE_OPERATING_PLAYBOOK.md): milestone-level operating emphasis

## Working Rule

These docs define how work is framed, gated, verified, and closed.

They do not replace the project-specific architecture, roadmap, or backlog.
