# Framework Docs

## Purpose

This folder contains the reusable workflow and governance spine of the framework kit.

These docs are meant to stay generic enough to reuse across projects.

## Suggested Reading Order

1. `DEVELOPMENT_FRAMEWORK.md`
2. `DECISION_GATES.md`
3. `PRE_PHASE_QA_PROTOCOL.md`
4. `PHASE_CLOSE_CHECKLIST.md`
5. `TEST_CHARTER.md`
6. `DONOR_TRIAGE_CHECKLIST.md`
7. `AGENT_ALIGNMENT_RULE.md`
8. `PHASE_OPERATING_PLAYBOOK.md`

## Working Rule

These docs define how work is framed, gated, verified, and closed.

They do not replace the project-specific architecture, roadmap, or backlog.
