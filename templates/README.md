# Template Guide

This folder contains the project-specific starter templates that pair with the reusable framework docs in [../docs/](../docs/README.md).

Use these files to define what your project actually is. The framework docs tell a repo how to operate. These templates tell the repo what it is building.

## Suggested Adoption Order

1. [STACK.template.md](STACK.template.md)
2. [SYSTEM_BOUNDARIES.template.md](SYSTEM_BOUNDARIES.template.md)
3. [ARCHITECTURE.template.md](ARCHITECTURE.template.md)
4. [ROADMAP.template.md](ROADMAP.template.md)
5. [BACKLOG.template.md](BACKLOG.template.md)

## File Guide

- [STACK.template.md](STACK.template.md): lock the implementation stack and rejected alternatives
- [SYSTEM_BOUNDARIES.template.md](SYSTEM_BOUNDARIES.template.md): define scope, non-goals, and trust boundaries
- [ARCHITECTURE.template.md](ARCHITECTURE.template.md): define modules, layers, runtime wiring, and testing seams
- [ROADMAP.template.md](ROADMAP.template.md): sequence milestones by risk and proof
- [BACKLOG.template.md](BACKLOG.template.md): break the next body of work into bounded tickets with explicit autonomy and gate fields

## Working Rule

Fill these in before asking agents to do meaningful implementation work.
